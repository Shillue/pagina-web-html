Entendendo Comunicação Client x Server

Voltar

Antes da internet colorida e cheia de memes, existia uma rede feita de terminais, fios e ideias geniais. A história da Web começa com os primeiros computadores, grandes como geladeiras, e mentes como as de Tim Berners-Lee, que pavimentaram o caminho para a internet atual.

Atualmente a comunicação digital acontece entre dois personagens principais: o client e o server. O client é quem consome o conteúdo — navegadores, aplicativos e dispositivos móveis — e o server é quem entrega esse conteúdo, seja ele um site, uma imagem ou um banco de dados.

Servidor web é um software responsável por aceitar pedidos em HTTP de clientes, geralmente os navegadores, e servi-los com respostas em HTTP, incluindo opcionalmente dados, que geralmente são páginas web, tais como documentos em HTML com objetos embutidos (imagens, etc) ou um computador que executa um programa que provê a funcionalidade descrita anteriormente. O mais popular, e mais utilizado no mundo, é o servidor Apache (software livre). A Microsoft possui a sua própria solução denominada IIS.

Sendo assim, os servidores funcionam como bibliotecas digitais gigantes, compostos de hardware e software especializados, e podem ser dedicados, compartilhados ou na nuvem. E para essa comunicação funcionar, entram as linguagens de programação, divididas entre client-side e server-side. Importante lembrar: HTML é uma linguagem de programação HTML não é linguagem de programação.

Introdução ao HTML na Prática

Voltar

O HTML é o esqueleto da web. Ele estrutura o conteúdo, dá ordem e faz tudo ganhar forma. Começando pela estrutura básica de um documento HTML: <html>, <head> e <body>

Ferramentas como o Inspetor de Elementos dos navegadores permitem visualizar e testar modificações diretamente nas páginas. Tags como <p>, <h1> até <h6>, listas ordenadas <ol> e não ordenadas <ul>, e links <a> são a base para montar um site funcional. Além disso, atributos nas tags como href, class, id oferecem mais controle sobre o comportamento e a apresentação dos elementos.

Trabalhando com Formulários em HTML

Voltar

Um formulárioem uma página web permite que um usuário insira dados que são enviados a um servidor para processamento. Desse modo, os formulários são as portas de entrada para interatividade com o usuário. Ao usar a tag <form>, é possível criar interfaces que coletam dados e os enviam para o servidor.

Sendo assim, existem vários tipos de <input>, como texto, senha, e-mail, botão,checkbox e radio, cada um ideal para uma finalidade específica. Entretanto, por ser auxiliar na semântica, além de oferece maior flexibilidade e controle sobre o estilo e comportamento a tag para Botões mais utilizada é <button> em vez <input type="button" value="Clique aqui">que acionam envios ou scripts, enquanto menus suspensos <select> oferecem múltiplas escolhas. Para textos mais longos, entra em cena a <textarea>. Cada elemento contribui para uma experiência mais fluida e funcional na hora de preencher cadastros, fazer buscas ou enviar comentários.

Estruturando seu HTML + Formatações

Voltar

Organizar conteúdo em HTML é torna a experiência de leitura mais fácil e mais agradável. Tags como <div> e <span> permitem agrupar e estilizar seções específicas.Para formular agrupamentos de campos de formulário com rótulos, entra o <fieldset>.

Além disso, recursos como <iframe> e <embed> permitem incorporar outros conteúdos, como vídeos do YouTube ou mapas. O uso correto de cores, seja por nomes, RGB ou hexadecimal, também faz parte da construção de uma interface mais acessível e visualmente agradável.

Trabalhando com Mídias utilizando HTML

Voltar

A linguagem HTML permite a inclusão de diversos tipos de mídias em páginas da web, como imagens, sons, vídeos e conteúdos incorporados. Para isso, utilizamos algumas tags específicas que facilitam essa integração e tornam a experiência do usuário mais rica e interativa.

O que são tags de mídias

Tags de mídias em HTML são elementos usados para inserir conteúdos visuais ou sonoros, como imagens, vídeos e áudios. Elas possibilitam a exibição desses recursos diretamente no navegador, sem a necessidade de plugins externos.

A tag <img>

A tag <img> é utilizada para exibir imagens. Possui os atributos src: que é o atributo mais importante e obrigatório, já que epecifica o caminho ou URL da imagem que deve ser exibida, alt: define um texto alternativo para a imagem, que é exibido caso a imagem não possa ser carregada ou se o usuário estiver usando um leitor de tela. O texto alternativo é essencial para acessibilidade

Exemplo: <img src="imagem.jpg" alt="Descrição da imagem">

Exemplos de formatos suportados
  • JPEG
  • PNG
  • GIF
  • SVG (gráfico vetorial)

A tag <audio>

Com a tag <audio>, é possível adicionar sons ou músicas a uma página. Ela pode incluir controles para o usuário iniciar ou pausar a reprodução. 


                <audio controls>
                    <source src="musica.mp3" type="audio/mpeg">
                    Seu navegador não suporta áudio.
                </audio>
Principais atributos
  • controls: exibe os controles padrão
  • autoplay: inicia a reprodução automaticamente
  • loop: repete o áudio
Observação

Permite múltiplas fontes de áudio dentro da mesma tag, e o navegador escolhe automaticamente a primeira que for compatível. Isso é útil porque nem todos os navegadores suportam todos os formatos de audio.

A tag <video>

A tag <video> é usada para incorporar vídeos. Assim como a tag de áudio, ela pode ter controles. 


                <video width="320" height="240" controls>
                    <source src="video.mp4" type="video/mp4">
                    Seu navegador não suporta vídeos.
                </video>
Recursos adicionais
  • Legendas
  • Vídeos alternativos
Observação

Permite múltiplas fontes de vídeo dentro da mesma tag, e o navegador escolhe automaticamente a primeira que for compatível. Isso é útil porque nem todos os navegadores suportam todos os formatos de vídeo. Por exemplo, o Safari prefere .mp4, enquanto navegadores como Firefox e Chrome também suportam .webm

A tag <track>

A tag <track> é usada dentro da tag <video> ou <audio> para adicionar legendas, descrições ou traduções.


                <video controls>
                    <source src="filme.mp4" type="video/mp4">
                    <track src="legenda.vtt" kind="subtitles" srclang="pt"       label="Português">
                </video>
Tipos de kind
  • subtitles: legendas
  • captions: legendas para deficientes auditivos
  • descriptions: descrições de áudio

A tag <iframe>

A tag <iframe> permite incorporar conteúdos de outras páginas, como vídeos do YouTube, mapas, entre outros.


                <iframe src="https://www.youtube.com/embed/ID_DO_VIDEO" width="560" height="315" allowfullscreen></iframe>
Uso comum
  • Vídeos de plataformas externas
  • Mapas - Google Maps
  • Documentos PDF

Criando Tabelas com HTML

Voltar

Tabelas são ideais para apresentar dados organizados. A estrutura começa com <table> e é composta por linhas <tr>, colunas de dados <td> e cabeçalhos <th>.

Para melhorar a clareza, usam-se <thead>, <tbody> e <tfoot> — que separam a tabela em seções. A estilização das tabelas com CSS ajuda na visualização e torna os dados mais agradáveis ao olhar, seja numa tabela de preços ou numa planilha de horários.

Entendendo HTML Semântico

Voltar
A semântica é o que atribui significado à linguagem HTML, formada por códigos que estruturam os elementos das páginas. Dessa forma, a conexão entre o design gráfico e as tags semânticas facilita a interação do usuário e do navegador com o site.

HTML semântico dá significado real aos elementos. Ao usar tags como <header>, <main>, <article>, <nav> ou <footer>, o navegador e os mecanismos de busca entendem melhor o papel de cada parte do conteúdo. Além de deixar o código mais organizado, isso melhora a acessibilidade para leitores de tela e contribui com o SEO é o conjunto de estratégias de otimização para mecanismos de busca, facilitando que buscadores como o Google encontrem e classifiquem melhor a página.

O HTML5 trouxe essas melhorias, além de elementos como <section>, <side>, <figure> e <figcaption>, ajudando a deixar o conteúdo mais claro, tanto para máquinas quanto para humanos. E para quem gosta de explorar dados, o web scraping se beneficia bastante de páginas bem estruturadas semânticamente.
